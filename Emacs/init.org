* このファイルについて

このファイルは [[https://orgmode.org/worg/org-contrib/babel/intro.html][Org Babel のリテラルプログラミング]] によって Emacs の設定を管理している。
=make link-emacs= を実行すると =init.org= がタングル（tangle）され、
=~/.emacs.d/init.el= と =~/.emacs.d/early-init.el= が生成される。

*編集するファイル*: =Emacs/init.org= のみ。=init.el= / =early-init.el= は自動生成のため直接編集しない。

*生成コマンド*:
#+begin_example
make link-emacs   # init.org → ~/.emacs.d/init.el と early-init.el を生成
#+end_example

*パッケージ構成の概要*:
| 役割 | パッケージ |
|------|-----------|
| パッケージ管理 | leaf, leaf-keywords |
| キーバインド | evil, evil-collection |
| 補完 | vertico, corfu, consult, orderless |
| LSP | eglot |
| 日本語入力 | ddskk (AZIK) |
| ターミナル | eat |
| Git | magit, git-gutter |

パッケージシステムの初期化より前に実行されるため、フレーム外観やネイティブコンパイルの設定に使う。

** フレームサイズ・外観

  				  (top . 30) ; 配置上

*** フレーム透明度の調整キーバインド

選択中フレームの不透明度 (=alpha=) を増減する。=C-c == で濃く (不透明)、
=C-c -= で薄く (透明)、=C-c 0= で既定値 (90) に戻す。=set-frame-parameter nil=
で選択フレームのみに作用するため、emacsclient で複数フレームを開いていても
フレーム単位で調整できる。

#+begin_src emacs-lisp :tangle yes
(defun my/frame-alpha--current ()
  "選択中フレームの不透明度 (0-100) を返す。未設定なら 100。"
  (let ((a (frame-parameter nil 'alpha)))
    (cond ((numberp a) a)
          ((consp a) (car a))            ; (active . inactive)
          (t 100))))

(defun my/frame-alpha-adjust (delta)
  "選択中フレームの不透明度を DELTA だけ変える (10-100 にクランプ)。"
  (let ((new (max 10 (min 100 (+ (my/frame-alpha--current) delta)))))
    (set-frame-parameter nil 'alpha new)
    (message "Frame opacity: %d%%" new)))

(defun my/frame-alpha-increase ()
  "背景を濃く (不透明に) する。"
  (interactive)
  (my/frame-alpha-adjust 5))

(defun my/frame-alpha-decrease ()
  "背景を薄く (透明に) する。"
  (interactive)
  (my/frame-alpha-adjust -5))

(defun my/frame-alpha-reset ()
  "不透明度を既定値 (90) に戻す。"
  (interactive)
  (set-frame-parameter nil 'alpha 90)
  (message "Frame opacity: 90%% (reset)"))

(global-set-key (kbd "C-c =") #'my/frame-alpha-increase)
(global-set-key (kbd "C-c -") #'my/frame-alpha-decrease)
(global-set-key (kbd "C-c 0") #'my/frame-alpha-reset)
#+end_src

*** 文字サイズの拡大・縮小キーバインド

=text-scale= による文字 (フォント) サイズの拡大・縮小を単打で行う。バッファ単位。
=C-= / C-+= で拡大、=C--= で縮小、=C-0= で既定サイズに戻す。
標準の =C-x C-+= / =C-x C--= / =C-x C-0= も引き続き使える。

衝突確認: =C-= / C-+= は未割当で安全。=C--= は標準の =negative-argument=、
=C-0= は =digit-argument= を上書きする (Evil 各ステートとの衝突はなし)。
数引数は =M--= / =M-0=〜=M-9= または =C-u= で代替できる。

#+begin_src emacs-lisp :tangle yes
(defun my/text-scale-reset ()
  "文字サイズを既定に戻す。"
  (interactive)
  (text-scale-set 0))

(global-set-key (kbd "C-=") #'text-scale-increase)
(global-set-key (kbd "C-+") #'text-scale-increase)
(global-set-key (kbd "C--") #'text-scale-decrease)
(global-set-key (kbd "C-0") #'my/text-scale-reset)
#+end_src

** パッケージマネージャ

[[https://github.com/conao3/leaf.el][leaf]] を使ってパッケージを宣言的に管理する。
=use-package= に近い DSL だが、キーワードが明確に分離されており読みやすい。

(eval-and-compile
  (customize-set-variable
   'package-archives '(("gnu" . "https://elpa.gnu.org/packages/")
      		("melpa" . "https://melpa.org/packages/")
      		("org" . "https://orgmode.org/elpa/")
      		("nongnu" . "https://elpa.nongnu.org/nongnu/")))
  (package-initialize)
  (use-package leaf :ensure t)

(leaf leaf-keywords
  :ensure t
  :init
  (leaf blackout :ensure t)
  :config
  ;; optional パッケージ
  (leaf hydra :ensure t)
  (leaf el-get :ensure t)
  (leaf blackout :ensure t)

  ;; leaf-keywordsの初期化
  (leaf-keywords-init)))

(leaf leaf-convert

  :ensure t)

** mise パスの設定

mise (asdf 互換バージョンマネージャ) の shims パスを Emacs の PATH に追加する。

#+begin_src emacs-lisp :tangle yes
;; mise (asdf-compatible version manager) のshimsパスをEmacsのPATHに追加
;; これにより terraform など mise 管理のツールを Emacs から利用できる
(let ((mise-shims (expand-file-name "~/.local/share/mise/shims")))
  (when (file-directory-p mise-shims)
    (add-to-list 'exec-path mise-shims)
    (setenv "PATH" (concat mise-shims ":" (getenv "PATH")))))
#+end_src

** Emacs 32 開発版バグの回避

Emacs 32.0.50 開発版で ~connection-local-set-profile-variables~ が ~(setting-constant nil)~ エラーを起こす問題のワークアラウンド。
~C-;~ (consult-buffer) や ~C-x C-b~ でバッファを開こうとした際、~project-try-vc~ 経由で ~vc-git~ が初めてロードされるときに発生する。

#+begin_src emacs-lisp :tangle yes
;; Workaround: Emacs 32 development build fails with (setting-constant nil) in
;; connection-local-set-profile-variables when vc-git loads interactively via
;; project-try-vc. The bug occurs in the gv/setf expansion for alist-get when
;; adding a new key. This advice catches the error and falls back to direct
;; alist manipulation.
(defvar vc-git--program-version nil)
(with-eval-after-load 'files-x
  (advice-add 'connection-local-set-profile-variables :around
    (lambda (orig-fn profile variables)
      (condition-case nil
          (funcall orig-fn profile variables)
        (setting-constant
         (let ((existing (assq profile connection-local-profile-alist)))
           (if existing
               (setcdr existing variables)
             (push (cons profile variables) connection-local-profile-alist))))))
    '((name . "emacs32-dev-fix"))))
#+end_src

*** システム情報表示 (CPU / メモリ / ディスク)

モードラインに macOS のシステム情報を表示する。
非同期シェルコマンドで 10 秒ごとに収集し、doom-modeline セグメントに反映する。

| 項目 | コマンド | 表示例 |
|------|---------|--------|
| CPU | =top -l 1= の user+sys | =CPU:23%= |
| メモリ | =vm_stat= から使用率計算 | =MEM:61%= |
| ディスク | =df -H /= の使用率 | =DSK:45%= |

#+begin_src emacs-lisp :tangle yes
(defvar my/sysinfo-string "" "モードラインに表示するシステム情報文字列.")

(defun my/sysinfo--shell-command ()
  "システム情報を収集するシェルコマンドを返す."
  (concat
   ;; CPU: top の1サンプルから user+sys を合算 (変数に取り出して % を除去)
   "cpu=$(top -l 1 -n 0 -s 0 2>/dev/null"
   " | awk '/CPU usage/{u=$3; s=$5; gsub(/%/,\"\",u); gsub(/%/,\"\",s); printf \"%.0f\", u+s}');"
   ;; メモリ: vm_stat から (active+wired+compressor)/total を計算
   "mem=$(vm_stat 2>/dev/null"
   " | awk '/Pages free:/{f=int($3)} /Pages active:/{a=int($3)}"
   " /Pages inactive:/{i=int($3)} /Pages wired down:/{w=int($4)}"
   " /Pages occupied by compressor:/{c=int($5)}"
   " END{t=f+a+i+w+c; if(t>0) printf \"%.0f\",(a+w+c)*100/t; else print \"?\"}');"
   ;; ディスク: 使用量(使用率) のみ — 分母(総量)は表示しない
   "disk=$(df -H / 2>/dev/null | awk 'NR==2{printf \"%s(%s)\",$3,$5}');"
   "printf 'CPU:%s%% | MEM:%s%% | DSK:%s' \"${cpu:-?}\" \"${mem:-?}\" \"${disk:-?}\""))

(defun my/sysinfo-update ()
  "システム情報を非同期で更新する."
  (let* ((buf (generate-new-buffer " *my-sysinfo*"))
         (proc (start-process "my-sysinfo" buf
                              "/bin/bash" "-c"
                              (my/sysinfo--shell-command))))
    (set-process-sentinel
     proc
     (lambda (p _e)
       (when (zerop (process-exit-status p))
         (setq my/sysinfo-string
               ;; mode-line format では % が制御文字になるため %% にエスケープ
               (replace-regexp-in-string
                "%" "%%"
                (string-trim
                 (with-current-buffer (process-buffer p)
                   (buffer-string)))))
         (force-mode-line-update t))

** クリップボード連携

ターミナル版 Emacs（=emacs -nw= / =emacsclient -nw=）には macOS の
システムクリップボードとの連携が標準では存在しない。そのため
=C-y= は Emacs 内部の kill-ring からしか取り出せず、ターミナルや
他アプリでコピーした内容と一致しない。

macOS の =pbcopy= / =pbpaste= を =interprogram-cut-function= /
=interprogram-paste-function= に割り当てて kill-ring とシステム
クリップボードを双方向に同期する。これにより =C-y=（=yank= /
=isearch-yank-kill=）が常にシステムクリップボードを反映する。

GUI フレームでは Emacs が標準でクリップボードを扱うため設定しない。

#+begin_src emacs-lisp :tangle yes

           (not (display-graphic-p))
           (executable-find "pbcopy"))

    "TEXT を macOS のシステムクリップボードへ送る."
    (let ((process-connection-type nil))
      (let ((proc (start-process "pbcopy" nil "pbcopy")))
        (process-send-string proc text)
        (process-send-eof proc))))
  (defun my/pbpaste ()
    "macOS のシステムクリップボードの内容を返す."
    (shell-command-to-string "pbpaste"))

  (setq interprogram-paste-function #'my/pbpaste))

SKK は ▽/▼ をバッファ上の実文字として挿入し、それを basis に
=skk-henkan-mode= 等の internal state を持つ。=C-h= や =C-w=、undo、yank
などの編集操作で ▽/▼ がバッファから消えると state だけ残り、
次の入力で次のいずれかのエラーで入力が完全に止まる:

- =既に▽モードに入っています= (skk-set-henkan-point-subr 由来)
- =▽がありません= (skk-change-marker 由来)

evil-insert-state-exit-hook は state 遷移時にしか発火しないため、
insert state 内で完結する編集ではこの不整合を拾えない。脱出弁として、
これらのエラーが発生した瞬間に SKK の internal state を強制リセットし、
=Backtrace= ではなくミニバッファのメッセージで再入力を促す。

#+begin_src emacs-lisp :tangle yes
(with-eval-after-load 'skk
  (defun my/skk-recover-on-state-mismatch (orig-fun &rest args)
    "ORIG-FUN 実行中に ▽/▼ マーカー不整合エラーが起きたら状態を復旧する."
    (condition-case err
        (apply orig-fun args)
      (error
       (let ((msg (error-message-string err)))
         (if (string-match-p
              "\\(?:▽\\|▼\\)がありません\\|既に▽モードに入っています"
              msg)
             (progn
               (setq skk-henkan-mode nil
                     skk-okuri-char nil
                     skk-okurigana nil
                     skk-henkan-key nil
                     skk-henkan-list nil
                     skk-henkan-count 0)
               (when (and (boundp 'skk-henkan-start-point)
                          (markerp skk-henkan-start-point))
                 (set-marker skk-henkan-start-point nil))
               (when (and (boundp 'skk-henkan-end-point)
                          (markerp skk-henkan-end-point))
                 (set-marker skk-henkan-end-point nil))
               (message "SKK: ▽/▼ 状態の不整合を復旧しました。もう一度入力してください。"))
           (signal (car err) (cdr err)))))))

  (dolist (fn '(skk-start-henkan skk-abbrev-mode skk-insert))
    (advice-add fn :around #'my/skk-recover-on-state-mismatch)))
#+end_src

*** スマート改行（括弧展開）

括弧の内側で Enter を押したとき、VSCode や JetBrains と同様に括弧を展開してインデントする。

例：
#+begin_example
// 入力前

  |
}
#+end_example

対応する括弧ペア: ={}=, =()=, =[]=。prog-mode の全バッファで有効。

#+begin_src emacs-lisp :tangle yes
(defun my/newline-and-indent-pair ()
  "括弧の間でEnterを押したとき、括弧を展開してインデントする。"
  (interactive)
  (let ((close (char-after)))
    (if (member (list (char-before) close)
                '((?{ ?}) (?\( ?\)) (?\[ ?\])))
        (progn
          (delete-char 1)
          (newline-and-indent)
          (let ((pos (point)))
            (newline)
            (insert close)
            (indent-according-to-mode)
            (goto-char pos)))
      (newline-and-indent))))

(add-hook 'prog-mode-hook
          (lambda ()
            (local-set-key (kbd "RET") #'my/newline-and-indent-pair)
            (when (fboundp 'evil-local-set-key)
              (evil-local-set-key 'insert (kbd "RET") #'my/newline-and-indent-pair))))

(defun my/org-slugify (str)
  "STR をファイル名に使える文字列に変換する."
  (let* ((s (string-trim str))
         (s (replace-regexp-in-string "\\s-+" "_" s))
         (s (replace-regexp-in-string "[/\\\\:*?\"<>|]" "" s)))
    (if (string-empty-p s) "untitled" s)))

(defun my/org-file-in-dir (dir label filetag headings)
  "DIR 内の org ファイルを completing-read で選択、または新規作成して返す.
LABEL はプロンプト文字列、FILETAG は #+filetags の値、
HEADINGS は新規作成時に挿入するトップレベル見出しのリスト。
既存ファイルは file-name-base で表示し、選択したパスを返す。
'[+ 新規作成]' を選ぶと名前を入力してファイルを作成する。"
  (let* ((dir   (expand-file-name dir))
         (_     (make-directory dir t))
         (files (directory-files dir nil "\\.org\\'" t))
         (new   "[+ 新規作成]")
         (sel   (completing-read
                 (format "%s: " label)
                 (cons new (mapcar #'file-name-base files))
                 nil nil))
         (path  (if (string= sel new)
                    (let* ((name (read-string "名前: "))
                           (slug (my/org-slugify name))
                           (p    (expand-file-name (concat slug ".org") dir)))
                      (unless (file-exists-p p)
                        (with-temp-file p
                          (insert (format "#+title: %s\n#+filetags: :%s:\n\n"
                                          name filetag))
                          (dolist (h headings)
                            (insert (format "* %s\n\n" h)))))
                      p)
                  (expand-file-name (concat sel ".org") dir))))
    path))

(defun my/org-sync ()
  "~/org を GitHub へ非同期で同期する."
  (interactive)
  (let ((script (expand-file-name "~/.conf/scripts/org-sync.sh")))
    (if (not (file-executable-p script))
        (message "org-sync: script not found: %s" script)
      (message "org sync: starting...")
      (set-process-sentinel
       (start-process "org-sync" "*org-sync*" script)
       (lambda (_proc event)
         (cond
          ((string-match "finished" event) (message "org sync: done"))
          ((string-match "exited abnormally" event)
           (message "org sync: failed — see *org-sync* buffer"))))))))

(defun my/org-sync-debounced ()
  "保存から 5 秒後に org sync を実行する（連続保存をまとめる）."
  (when my/org-sync-timer
    (cancel-timer my/org-sync-timer))
  (setq my/org-sync-timer
        (run-with-timer 5 nil #'my/org-sync)))

  "アジェンダの日付行から daily_YYYY.org の当該日見出しへ移動する."
  (interactive)
  (let* ((line (buffer-substring-no-properties
                (line-beginning-position) (line-end-position)))
         (month-alist '(("January" . 1) ("February" . 2) ("March" . 3)
                        ("April" . 4) ("May" . 5) ("June" . 6)
                        ("July" . 7) ("August" . 8) ("September" . 9)
                        ("October" . 10) ("November" . 11) ("December" . 12))))
    (if (string-match
         (concat "[A-Za-z]+\\s-+\\([0-9]+\\)\\s-+"
                 "\\(January\\|February\\|March\\|April\\|May\\|June\\|"
                 "July\\|August\\|September\\|October\\|November\\|December\\)"
                 "\\s-+\\([0-9]\\{4\\}\\)")
         line)
        (let* ((day   (string-to-number (match-string 1 line)))
               (month (cdr (assoc (match-string 2 line) month-alist)))
               (year  (string-to-number (match-string 3 line)))
               (daily-file (expand-file-name
                            (format "personal/daily_%04d.org" year)
                            "~/org")))
          (unless (file-exists-p daily-file)
            (with-temp-file daily-file
              (insert (format "#+title: Daily %d\n#+filetags: :personal:daily:\n\n" year))))
          (find-file daily-file)
          (goto-char (point-min))
          (if (re-search-forward
               (format "^\\*\\{3\\}\\s-+%04d-%02d-%02d" year month day)
               nil t)
              (progn
                (org-reveal)
                (recenter)
                (message "%04d-%02d-%02d の日記を開きました" year month day))
            (message "%04d-%02d-%02d の日記がありません。C-c c d で作成してください。"
                     year month day)))
      (message "この行に日付が見つかりません。"))))

(defun my/org-agenda-goto-or-daily ()

  (interactive)
  (if (org-get-at-bol 'org-marker)
      (org-agenda-goto)
    (my/org-agenda-open-daily-at-point)))

*** Markdown

(defvar-local my/markdown-preview--last-html nil
  "直近に生成したプレビュー HTML。再生成時に削除して /tmp の増殖を防ぐ。")

(defun my/markdown-preview--render (buf)
  "BUF のマークダウンを GitHub 風 HTML に変換し、ファイルパスを返す."
  (let* ((md-tmp (unless (buffer-file-name buf)
                   (let ((f (make-temp-file "md-src-" nil ".md")))
                     (with-current-buffer buf
                       (write-region (point-min) (point-max) f))
                     f)))
         (md-file (or (buffer-file-name buf) md-tmp))
         (html-file (make-temp-file "md-preview-" nil ".html"))
         (css-file (make-temp-file "md-css-" nil ".css"))
         (header-file (make-temp-file "md-mermaid-" nil ".html")))
    (write-region my/markdown-preview-css nil css-file)
    (write-region my/markdown-preview-mermaid-header nil header-file)
    (unless (= 0 (call-process "pandoc" nil nil nil
                               "-f" "gfm"
                               "-t" "html5"
                               "--standalone"
                               "--embed-resources"
                               "--syntax-highlighting=tango"
                               (format "--css=%s" css-file)
                               (format "--include-in-header=%s" header-file)
                               "-o" html-file
                               md-file))
      (delete-file css-file)
      (delete-file header-file)
      (when md-tmp (delete-file md-tmp))
      (delete-file html-file)
      (error "pandoc の実行に失敗しました"))
    (delete-file css-file)
    (delete-file header-file)
    (when md-tmp (delete-file md-tmp))
    ;; 前回のプレビュー HTML を破棄（毎回ユニークな URL にして WebKit に再読込させる）
    (with-current-buffer buf
      (when (and my/markdown-preview--last-html
                 (file-exists-p my/markdown-preview--last-html))
        (ignore-errors (delete-file my/markdown-preview--last-html)))
      (setq my/markdown-preview--last-html html-file))
    html-file))

(defun my/markdown-preview (&optional select)
  "現在のマークダウンバッファを WebKit (xwidget) でブラウザのようにプレビューする.
pandoc で GitHub 風 HTML に変換し、xwidget-webkit で描画する。
インタラクティブ呼び出し時はプレビュー window を選択する。"
  (interactive (list t))
  (unless (and (display-graphic-p) (featurep 'xwidget-internal))
    (user-error "WebKit プレビューには xwidget 対応の GUI Emacs が必要です"))
  (require 'xwidget)
  (let* ((src-buf (current-buffer))
         (html-file (my/markdown-preview--render src-buf))
         (url (concat "file://" html-file)))
    ;; 既存の WebKit セッションを再利用しつつ URL をロード（編集中の window 構成は保持）
    (save-window-excursion
      (xwidget-webkit-browse-url url))
    (let* ((session (xwidget-webkit-current-session))
           (buf (and session (xwidget-buffer session))))
      (unless (buffer-live-p buf)
        (error "WebKit セッションの取得に失敗しました"))
      (if select
          (pop-to-buffer buf)
        (display-buffer buf)))))

*** Web 検索

=C-c w= で Google 検索する。結果は xwidget-webkit (WebKit) で Emacs 内に
ブラウザ同様にリッチ表示する。リージョン選択中はその文字列、それ以外は

#+begin_src emacs-lisp :tangle yes
(defun my/web-search (query)
  "QUERY を Google で検索し、結果を xwidget-webkit (WebKit) で Emacs 内に表示する.
リージョン選択中はその文字列、それ以外は point のシンボルを既定値にする。"
  (interactive
   (let ((default (if (use-region-p)
                      (buffer-substring-no-properties (region-beginning) (region-end))
                    (thing-at-point 'symbol t))))
     (list (read-string (format-prompt "Web 検索 (Google)" default) nil nil default))))
  (unless (and (display-graphic-p) (featurep 'xwidget-internal))
    (user-error "WebKit 表示には xwidget 対応の GUI Emacs が必要です"))
  (require 'xwidget)
  (let ((q (string-trim (or query ""))))
    (when (string-empty-p q)
      (user-error "検索語が空です"))
    (let ((url (concat "https://www.google.com/search?q=" (url-hexify-string q))))
      ;; 既存の WebKit セッションを再利用しつつ URL をロード（編集中の window 構成は保持）
      (save-window-excursion
        (xwidget-webkit-browse-url url))
      (let* ((session (xwidget-webkit-current-session))
             (buf (and session (xwidget-buffer session))))
        (unless (buffer-live-p buf)
          (error "WebKit セッションの取得に失敗しました"))
        (pop-to-buffer buf)))))

(global-set-key (kbd "C-c w") #'my/web-search)

;; --- Vimium 風リンクヒント (xwidget-webkit) ---
;; ページ内の <a href> を JS で抽出し、completing-read で選んで遷移する。
(defconst my/xwidget-webkit-collect-links-js
  "JSON.stringify(Array.prototype.slice.call(document.querySelectorAll('a[href]')).map(function(a){return {text:(a.innerText||a.textContent||'').replace(/\\s+/g,' ').trim().slice(0,120), url:a.href};}).filter(function(x){return x.url && x.url.indexOf('javascript:')!==0;}))"
  "現在の WebKit ページのリンクを JSON 文字列で返す JavaScript。")

(defun my/xwidget-webkit-hint--open (result)
  "RESULT (リンク配列の JSON 文字列) から completing-read で選んで遷移する."
  (let ((parsed (ignore-errors
                  (json-parse-string (or result "[]")
                                     :object-type 'alist :array-type 'list))))
    (unless (listp parsed)
      (setq parsed nil))
    (unless parsed
      (user-error "リンクが取得できませんでした"))
    (let ((seen (make-hash-table :test 'equal))
          (candidates nil))
      (dolist (link parsed)
        (let* ((text (or (alist-get 'text link) ""))
               (url (alist-get 'url link))
               (label (format "%s  —  %s"
                              (if (string-empty-p text) "(no text)" text)
                              url)))
          (when (and url (not (gethash label seen)))
            (puthash label url seen)
            (push (cons label url) candidates))))
      (setq candidates (nreverse candidates))
      (let* ((choice (completing-read "Link: " candidates nil t))
             (url (cdr (assoc choice candidates)))
             (session (xwidget-webkit-current-session)))
        (when (and url session)
          (xwidget-webkit-goto-uri session url))))))

(defun my/xwidget-webkit-hint ()
  "現在の WebKit ページのリンクを completing-read で選んで開く (Vimium 風)."
  (interactive)
  (let ((session (xwidget-webkit-current-session)))
    (unless session
      (user-error "WebKit セッションがありません"))
    (xwidget-webkit-execute-script
     session my/xwidget-webkit-collect-links-js #'my/xwidget-webkit-hint--open)))

;; --- ページ内検索 (xwidget-webkit) ---
;; macOS の xwidget ビルドでは組込みの xwidget-webkit-isearch-mode が
;; "Widget has no ongoing search operation" で失敗するため、ブラウザ標準の
;; window.find() を JS で呼ぶ自前検索に差し替える。
(defvar-local my/xwidget-webkit-search--last ""
  "直近のページ内検索語。")

(defun my/xwidget-webkit-search--find (query backward)
  "QUERY を window.find() で検索する。BACKWARD が非 nil なら後方検索。"
  (require 'json)
  (let ((session (xwidget-webkit-current-session)))
    (unless session
      (user-error "WebKit セッションがありません"))
    ;; window.find(string, caseSensitive, backwards, wrapAround, ...)
    (xwidget-webkit-execute-script
     session
     (format "window.find(%s, false, %s, true, false, false, false)"
             (json-encode-string query)
             (if backward "true" "false")))))

(defun my/xwidget-webkit-search (query)
  "ページ内をテキスト検索する (直近の検索語が既定値)。
同じ語のまま =RET= を繰り返すと次のヒットへ進む。"
  (interactive
   (list (read-string (format-prompt "ページ内検索" my/xwidget-webkit-search--last)
                      nil nil my/xwidget-webkit-search--last)))
  (let ((q (or query "")))
    (when (string-empty-p q)
      (user-error "検索語が空です"))
    (setq my/xwidget-webkit-search--last q)
    (my/xwidget-webkit-search--find q nil)))

(defun my/xwidget-webkit-search-previous ()
  "直近の検索語で前方向 (後方) のヒットへ移動する。"
  (interactive)
  (if (> (length my/xwidget-webkit-search--last) 0)
      (my/xwidget-webkit-search--find my/xwidget-webkit-search--last t)
    (call-interactively #'my/xwidget-webkit-search)))

(with-eval-after-load 'xwidget
